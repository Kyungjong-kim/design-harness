# 스킬 목록

> **이 목록은 제작자의 전체 Claude Code 환경 기준이다.**
> `install.sh`로 설치되는 번들 스킬은 **"하네스 번들 스킬"** 섹션만 해당된다.
> 상단의 "개발 도구"·"UI/디자인" 스킬은 별도 설치 항목이며 패키지에 포함되지 않는다.

`~/.claude/skills/` 에 설치된 스킬 전체 목록.

## 개발 도구

| 스킬 | 트리거 | 용도 |
|------|--------|------|
| `skill-creator` | 새 스킬 만들 때 | 스킬 생성·수정 가이드 (init / package / validate 스크립트 포함) |
| `executing-plans` | "플랜 실행", "/executing-plans" | `docs/plans/*.md` 구현 플랜을 태스크 단위로 순서대로 실행 |
| `writing-plans` | 스펙·요구사항 있을 때 코드 작성 전 | 다단계 태스크 구현 플랜 작성 |
| `webapp-testing` | 로컬 웹앱 검증 시 | Playwright로 브라우저 기능 테스트·스크린샷·로그 확인 |
| `explain-code` | "이 코드 설명해줘" | 다이어그램·유추를 사용한 코드 설명 |
| `mcp-builder` | MCP 서버 만들 때 | FastMCP(Python) / MCP SDK(TS) 기반 MCP 서버 생성 가이드 |

## UI / 디자인

| 스킬 | 트리거 | 용도 |
|------|--------|------|
| `frontend-design` | 웹 컴포넌트·페이지 구현 | 고품질 프론트엔드 UI 생성 (React, HTML/CSS) |
| `web-artifacts-builder` | 복잡한 멀티컴포넌트 아티팩트 | React + Tailwind + shadcn/ui 기반 복잡한 HTML 아티팩트 |
| `canvas-design` | 포스터·아트·디자인 생성 | PNG/PDF 시각 디자인 생성 |
| `theme-factory` | 아티팩트 스타일링 | 슬라이드·문서·HTML에 10개 프리셋 테마 적용 |
| `brand-guidelines` | 브랜드 가이드 적용 | 특정 브랜드 색상·타이포그래피 아티팩트에 적용 |
| `design-token-review` | 디자이너가 새 토큰 제시 시 | 디자인 토큰 변경·누락·추가 항목 파악 후 반영 방법 결정 |

## 하네스 번들 스킬

`install.sh` 실행 시 `~/.claude/skills/`에 복사되는 스킬 목록.

### 프로젝트 워크플로 (항상 설치)

`하네스/skills/` — 모든 환경에 기본 설치.

| 스킬 | 트리거 | 용도 |
|------|--------|------|
| `project-init` | "하네스 구축", "/project-init" | 코드베이스 분석 → CLAUDE.md + HANDOFF 3종 초기화 원스톱 |
| `project-fix` | "버그 수정", "/project-fix" | 이슈 URL/번호 → 서브이슈 생성 + 브랜치 준비 자동화 |
| `project-issue` | "이슈 생성", "/project-issue" | 상위+하위 GitHub 이슈 세트 인터랙티브 생성 |
| `project-pr` | "PR 만들어", "/project-pr" | 이슈 연결·Co-Authored-By 포함 PR 자동 생성 |
| `session-close` | "세션 종료", "/session-close" | HANDOFF 3종 갱신 + 세션_노트 prepend |
| `document-review` | "문서 검토", "/document-review" | 문서 구조·연동·완전성 검증 후 이슈 수정 |

### 개인 생산성 (선택 설치)

`하네스/personal-skills/` — `--personal` 플래그 또는 설치 중 y 응답 시 복사.

| 스킬 | 트리거 | 용도 |
|------|--------|------|
| `daily-note` | "오늘 노트", "/daily-note" | 일일노트 생성(아침) / git 커밋 기반 작업 로그 완성(저녁) |
| `standup` | "스탠드업", "/standup" | 어제 커밋 + 오늘 할 일 → 스탠드업 3항목 자동 구성 |
| `meeting-note` | "회의 정리", "/meeting-note" | 회의 내용 붙여넣기 → 논의/결정/액션아이템 구조화 저장 |
| `til` | "TIL", "/til" | 배운 것 → TIL 형식 구조화 저장 |
| `weekly-retro` | "주간 회고", "/weekly-retro" | 이번 주 커밋·TIL 취합 → KPT 회고 자동 생성 |
| `weekly-meeting-update` | "주간회의록", "/weekly-meeting-update" | 이번 주 커밋·HANDOFF 기반 주간회의록 섹션 자동 생성 |

> **노트 경로 설정**: `install.sh --personal` 실행 중 자동으로 경로를 묻고 치환한다. 설치 시 건너뛴 경우 `Claude Code/트러블슈팅.md` → "개인 스킬 노트 경로 미설정" 참조.

## 스킬 관리 명령어

```bash
# 스킬 초기화
python3 ~/.claude/skills/skill-creator/scripts/init_skill.py <스킬명> --path ~/.claude/skills/

# 스킬 패키징 (.skill 파일 생성)
python3 ~/.claude/skills/skill-creator/scripts/package_skill.py ~/.claude/skills/<스킬명>

# 유효성 검사
python3 ~/.claude/skills/skill-creator/scripts/quick_validate.py ~/.claude/skills/<스킬명>
```

> `.skill` 파일은 로컬 전용 — 프로젝트 `.gitignore`에 `*.skill` 추가 필수.
