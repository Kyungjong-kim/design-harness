# 트러블슈팅

하네스 설치·운영 중 자주 발생하는 문제와 해결 방법을 정리한다.

---

## 설치

### install.sh 실행 권한 오류

```
zsh: permission denied: ./install.sh
```

**원인:** 실행 권한 없음.

```bash
chmod +x install.sh
bash install.sh
```

---

### install.sh 중간 실패 후 재실행

설치 도중 실패했거나 일부 옵션만 다시 실행하고 싶을 때.

```bash
# 훅만 재설치
bash install.sh --hooks

# 개인 스킬만 재설치
bash install.sh --personal

# 전체 재설치 (기존 파일 강제 덮어씀)
bash install.sh --force --personal --hooks
```

> `--force` 재실행 시 `~/.claude/skills/` 아래 파일이 덮어써진다.  
> 스킬 파일을 직접 수정했다면 백업 후 실행할 것.

---

### gh auth 오류

```
gh: command not found
```

**원인:** GitHub CLI 미설치.

```bash
brew install gh
gh auth login
```

```
gh auth status → You are not logged into any GitHub hosts.
```

```bash
gh auth login
# 브라우저 인증 또는 Personal Access Token 입력
gh auth status   # 인증 확인
```

---

### 개인 스킬 노트 경로 미설정

`/daily-note`, `/standup` 등 실행 시 `<개인_노트_경로>` 문자열이 경로로 사용됨.

**확인:**
```bash
grep -l "<개인_노트_경로>" ~/.claude/skills/*/SKILL.md 2>/dev/null
```

**수동 설정:**
```bash
NOTE_PATH="/Users/<사용자명>/Documents/Dev-Vault"  # 실제 경로로 교체

for skill in daily-note standup meeting-note til weekly-retro weekly-meeting-update; do
  FILE=~/.claude/skills/$skill/SKILL.md
  [ -f "$FILE" ] && sed -i.bak "s|<개인_노트_경로>|$NOTE_PATH|g" "$FILE" && rm -f "${FILE}.bak" && echo "✓ $skill"
done
```

## 훅

### 훅이 동작하지 않음 (경고 메시지 미출력)

파일을 수정해도 `check-source-doc.sh` 경고가 나오지 않을 때.

**원인 1:** `~/.claude/settings.json`에 훅이 등록되지 않음.

```bash
cat ~/.claude/settings.json | python3 -c "import json,sys; d=json.load(sys.stdin); print(json.dumps(d.get('hooks', {}), indent=2))"
```

`PostToolUse` → `Edit|Write` 항목이 없으면 `install.sh --hooks` 재실행.

**원인 2:** 스크립트 실행 권한 없음.

```bash
ls -l ~/.claude/hooks/
chmod +x ~/.claude/hooks/check-source-doc.sh
chmod +x ~/.claude/hooks/pre-commit-doc-check.sh
```

---

### `hooks-config.sh` 미설정 — 플레이스홀더 경고 출력

소스 파일을 수정했을 때 아래 경고가 출력되는 경우:

```
⚠️  [훅 미설정] check-source-doc.sh의 [프로젝트 설정] 변수가 아직 교체되지 않았습니다.
```

**원인:** 레포 루트에 `.claude/hooks-config.sh`가 없거나 `FE_PATTERN` 변수가 미설정.

**해결:**

```bash
# 프로젝트 루트에서 /project-init 스킬 실행 → Phase 7-C 자동 생성
# 또는 수동 작성 (변수명은 훅_스크립트_전문.md § 신규 프로젝트 적용 절차 참조)
cat > .claude/hooks-config.sh << 'EOF'
# check-source-doc.sh 변수
FE_PATTERN="user-portal/frontend/(pages|components|hooks|api)/([^/]+)/"
FE_DOC_BASE="user-portal/docs/pages"
FE_PROJECT="user-portal"
# pre-commit-doc-check.sh 변수
FE_COMMIT_PATTERN="^user-portal/frontend/.*\.(ts|tsx)$"
FE_DOMAIN_EXTRACT='s|user-portal/frontend/pages/([^/]+)/.*|\1|'
FE_DOC_DIR_BASE="user-portal/docs/pages"
EOF
```

---

### git commit 전 훅 경고가 너무 빈번함

`pre-commit-doc-check.sh`가 관련 없는 파일 커밋에도 경고를 냄.

**원인:** `.claude/hooks-config.sh`의 `FE_COMMIT_PATTERN` / `BE_COMMIT_PATTERN` 범위가 너무 넓음.

```bash
# .claude/hooks-config.sh 패턴 좁히기
FE_COMMIT_PATTERN="user-portal/frontend/apps/client/src/**"
```

## `/project-init` 스킬

### Phase 7-C 실행 후 `hooks-config.sh` 미생성

**원인:** 소스 경로 분석에서 패턴을 감지하지 못한 경우.

**해결:** 프로젝트 루트에서 수동으로 `.claude/hooks-config.sh` 작성 후 `/project-init` Phase 10 진단 재실행.

---

### `/project-init` 재실행이 필요한 경우

| 케이스 | 조치 |
|--------|------|
| 소스 경로 구조 변경 | Phase 7-C 재실행 → `hooks-config.sh` 갱신 |
| 신규 팀원 온보딩 | Phase 1부터 전체 재실행 |
| 훅 동작 이상 | Phase 10 진단만 재실행 |
| CLAUDE.md 초기화 필요 | Phase 3 재실행 |

## HANDOFF 문서

### HANDOFF_NOW.md 60줄 초과

**처리:**
1. `§2` 완료 항목 즉시 삭제
2. 2주 이상 미착수 항목 → `⏸` 태그 추가 후 §2 하단으로 이동
3. 여전히 60줄 초과 → `⏸` 항목을 `HANDOFF.md`로 이관

자세한 기준: `운영_가이드/HANDOFF_유지_규칙.md` 참조.

## 기타

### 스킬이 목록에 없거나 실행 안 됨

`~/.claude/skills/<스킬명>/SKILL.md` 파일 존재 여부 확인:

```bash
ls ~/.claude/skills/
```

없으면 아래 명령으로 재설치:
- 프로젝트 스킬 (`project-init`, `session-close` 등): `bash install.sh`
- 개인 스킬 (`daily-note`, `standup` 등): `bash install.sh --personal`
