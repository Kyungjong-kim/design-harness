# BE 기능 정리 문서 템플릿

`docs/<서비스명>/pages/<도메인>/BE_<기능명>.md` 에 위치하는 기능 정리 문서 템플릿.
PostToolUse 훅이 `BE_*.md` 파일 존재 여부를 감지해 작성을 유도하는 문서.

> **에이전트 사용 안내**
> 1. 이 템플릿을 해당 경로에 복사한다.
>    - 예: `docs/<서비스명>/pages/<도메인>/BE_<기능명>.md`
> 2. `<!-- FILL: ... -->` 주석 안의 내용을 실제 구현 기반으로 채운다.
> 3. 파일명 규칙: `BE_` + 기능명 대문자 스네이크케이스 + `.md`
>    - 예: `BE_USER_AUTH.md`, `BE_CHAT_MESSAGE_API.md`
>
> **작성 기준**
> - 엔드포인트가 **무엇을 하는지** (목적·책임)
> - **요청/응답 스펙**이 무엇인지 (필드·타입·제약)
> - **비즈니스 로직**이 어디서 처리되는지 (service/handler 레이어)
> - **DB 변경**이 있는지 (마이그레이션 이력)
> - 알아두어야 할 **엣지케이스·주의사항**
>
> **작성 후 검증 순서** (참조: `하네스/운영_가이드/API_검증_전략.md`)
> 1. 단위 테스트: `pytest tests/ -v` / `go test ./...`
> 2. 통합 테스트: httpx TestClient / httptest로 엔드포인트 직접 호출
> 3. 실서버 확인 (배포 전): `curl localhost:<포트>/api/...`


```json
{
  "[필드명]": "[타입·설명]",

}
```

| 상태 코드 | 조건 | 응답 형식 |
