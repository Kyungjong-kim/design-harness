# 컴포넌트 정리 문서 템플릿 — 라이브러리 전용

`docs/<라이브러리명>/components/<kebab-case>.md` 에 위치하는 컴포넌트 정리 문서 템플릿.
앱 하네스의 `FE_기능정리_템플릿.md` 라이브러리 대체본.

> **작성 정책 — 강제 아님**
>
> 모든 컴포넌트마다 작성하지 않는다. 복잡한 컴포넌트만 작성한다.
>
> **작성 권장 기준**
> - DnD·차트·복합 폼 등 구현 자체가 복잡한 컴포넌트
> - Accordion·Dialog·Combobox 등 합성 컴포넌트(Trigger·Content 분리)
> - 외부 라이브러리 의존도가 높은 것 (Radix·dnd-kit·sonner 등)
> - 사용 시 자주 헷갈리는 prop·callback 또는 비제어/제어 모드가 있는 것
> - 접근성 키 조합·스크린리더 동작이 중요한 것
>
> **작성 생략 가능**
> - 단순 props만 받는 컴포넌트 (Spinner·Skeleton·Badge 등)
> - variant만 추가하는 변경 → Storybook 스토리·테스트 갱신으로 갈음
> - 1차 문서는 항상 **Storybook 스토리 + 단위 테스트**다. 본 문서는 보완용.

> **에이전트 사용 안내**
> 1. 이 템플릿을 해당 경로에 복사한다.
>    - 예: `docs/<라이브러리명>/components/dnd-list.md`
> 2. `<!-- FILL: ... -->` 주석 안의 내용을 실제 구현 기반으로 채운다.
> 3. 파일명 규칙: `<kebab-case>.md` (컴포넌트 디렉토리명과 일치)
> 4. 사용하지 않는 *(선택)* 섹션은 행·블록을 통째로 삭제한다.

## 4. 사용 예시

<!-- FILL: 실제 동작하는 사용 예시 코드 (Storybook Default 스토리에서 추출 가능)
  - 기본 사용
  - 합성 사용 (합성 컴포넌트일 때만)
  - 고급 사용 (제어 모드·콜백 사용 등이 있을 때만)

### 4-1. 기본 사용

```tsx
import { [ComponentName] } from "[라이브러리명]";

<[ComponentName] [prop1]={value1} [prop2]={value2}>
  [children]
</[ComponentName]>
```

### 4-2. 합성 사용 *(선택 — 합성 컴포넌트일 때)*

```tsx
import { [Component], [Component].Trigger, [Component].Content } from "[라이브러리명]";

<[Component]>
  <[Component].Trigger>열기</[Component].Trigger>
  <[Component].Content>
    [콘텐츠]
  </[Component].Content>
</[Component]>
```

### 4-3. 제어 모드 *(선택 — 비제어/제어 모드 모두 지원할 때)*

```tsx
const [open, setOpen] = useState(false);

<[Component] open={open} onOpenChange={setOpen}>
  ...
</[Component]>
```
