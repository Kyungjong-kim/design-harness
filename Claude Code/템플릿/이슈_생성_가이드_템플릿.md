> ⚠️ **하위 이슈 규칙**
> 1. **타입**: 상위 이슈에서 분리된 하위 이슈는 **반드시 `Task` 타입**으로 생성한다.
> 2. **서브이슈 등록**: 하위 이슈 생성 후 **반드시 상위 이슈의 서브이슈로 등록**한다.

---

## 하위 이슈 생성 및 등록 프로세스

### Step 1 — 하위 이슈 생성

```bash
gh issue create \
  --repo <org>/<repo> \
  --title "[FE] 하위 작업 제목" \
  --label "frontend" \
  --body "본문"
```

### Step 2 — 타입을 Task로 설정

```bash
CHILD_ISSUE_NUMBER=<번호>
CHILD_NODE_ID=$(gh api repos/<org>/<repo>/issues/$CHILD_ISSUE_NUMBER --jq '.node_id')

gh api graphql -f query="
mutation {
  updateIssue(input: {
    id: \"$CHILD_NODE_ID\",
    issueTypeId: \"<TASK_TYPE_NODE_ID>\"
  }) {
    issue { number title issueType { name } }
  }
}"
```

### Step 3 — 서브이슈로 등록

```bash
PARENT_ISSUE_NUMBER=<번호>
PARENT_NODE_ID=$(gh api repos/<org>/<repo>/issues/$PARENT_ISSUE_NUMBER --jq '.node_id')

gh api graphql -f query="
mutation {
  addSubIssue(input: {
    issueId: \"$PARENT_NODE_ID\",
    subIssueId: \"$CHILD_NODE_ID\"
  }) {
    issue { number title }
    subIssue { number title }
  }
}"
```

> **Claude Code 에이전트 규칙**: 하위 이슈 생성 시 Step 3 서브이슈 등록까지 자동 수행한다.

## 본문 템플릿

모든 이슈는 아래 **4개 섹션 고정 구조**를 사용한다.

```markdown
### 상세설명

<!-- 이슈 목적·범위를 2~4문장으로 요약 -->

## Background

<!-- 왜 이 작업이 필요한지, 현재 문제 또는 개선 동기 -->

## To Do

- [ ] 항목 1
- [ ] 항목 2

## See Also

- 관련 문서 또는 링크
```

### 타입별 작성 가이드

| 타입 | 상세설명 | Background | To Do | See Also |
|------|----------|------------|-------|----------|
| **Task** | 작업 목적·범위 | 요청 경위, 작업 배경 | 구체적인 작업 단계 | 관련 문서, 기존 이슈 |
| **Bug** | 버그 증상·영향 범위 | 재현 방법, 발생 환경 | 원인 분석→수정→테스트 | 에러 로그, 관련 PR |
| **Feature** | 추가 기능과 사용자 가치 | 기획 배경, 사용자 요구 | 구현 항목 체크리스트 | 기획서, 디자인 시안 |
| **Improvement** | 개선 대상·기대 효과 | 현재 문제점, 개선 필요성 | 개선 작업 단계 | 기술부채 문서, 코드 경로 |

## CLI 직접 생성 명령어

```bash
# 기본 생성
gh issue create \
  --repo <org>/<repo> \
  --title "[FE] 제목" \
  --label "enhancement" \
  --body "본문"

# 타입 설정 (GraphQL, 이슈 생성 후)
ISSUE_NODE_ID=$(gh api repos/<org>/<repo>/issues/<번호> --jq '.node_id')
gh api graphql -f query="
mutation {
  updateIssue(input: {
    id: \"$ISSUE_NODE_ID\",
    issueTypeId: \"<TYPE_NODE_ID>\"
  }) {
    issue { number title issueType { name } }
  }
}"
```

## 스킬 사용법

```
/project-issue
```
<!-- FILL: 프로젝트 커스텀 스킬명으로 교체 -->

스킬 실행 시 Claude가 아래 순서로 질문합니다:

1. **제목** — 접두사 포함 제목 입력
2. **타입** — Task / Bug / Feature / Improvement 선택
3. **라벨** — 목록에서 선택 (복수 가능)
4. **마일스톤** — 목록에서 선택
5. **본문** — 작업 개요 입력 (선택, 템플릿 자동 적용)
6. **담당자** — GitHub 사용자명 (선택)

확인 후 이슈를 생성하고 URL을 반환합니다.

## GitHub ISSUE_TEMPLATE (선택)

`.github/ISSUE_TEMPLATE/` 에 yml 폼 파일을 추가하면 GitHub 웹 UI에서도 구조화된 이슈 작성이 가능하다.

```
.github/ISSUE_TEMPLATE/
  bug_report.yml
  feature_request.yml
  config.yml          # 템플릿 선택 화면 설정
```

`config.yml` 예시:
```yaml
blank_issues_enabled: false
contact_links: []
```
