# FE 기능 정리 문서 템플릿

`docs/<서비스명>/pages/<도메인>/FE_<기능명>.md` 에 위치하는 기능 정리 문서 템플릿.
PostToolUse 훅이 `FE_*.md` 파일 존재 여부를 감지해 작성을 유도하는 문서.

> **에이전트 사용 안내**
> 1. 이 템플릿을 해당 경로에 복사한다.
>    - 예: `docs/<서비스명>/pages/<도메인>/FE_<기능명>.md`
> 2. `<!-- FILL: ... -->` 주석 안의 내용을 실제 구현 기반으로 채운다.
> 3. 파일명 규칙: `FE_` + 기능명 대문자 스네이크케이스 + `.md`
>    - 예: `FE_CHAT_MAIN_SCREEN.md`, `FE_AUTH_CALLBACK.md`
>
> **작성 기준**
> - 기능이 **어디서 렌더링**되는지
> - **노출 조건**이 무엇인지 (guard, 상태값, 권한)
> - **데이터 흐름**이 어떻게 되는지 (API 호출 → 상태 → UI 갱신)
> - 알아두어야 할 **엣지 케이스·주의사항**
