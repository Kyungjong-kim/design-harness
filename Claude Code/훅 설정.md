# 훅 설정

`~/.claude/settings.json`에 등록된 훅 목록과 동작 설명.

## 훅 이벤트 종류

| 이벤트 | 타이밍 |
|--------|--------|
| `PreToolUse` | 도구 실행 **전** |
| `PostToolUse` | 도구 실행 **후** (성공) |
| `PostToolUseFailure` | 도구 실행 **후** (실패) |
| `Stop` | Claude 응답 완료 후 |

## settings.json 기본 구조

```json
"hooks": {
  "PreToolUse": [
    {
      "matcher": "<도구명>",
      "hooks": [{ "type": "command", "command": "bash ~/.claude/hooks/<스크립트>.sh" }]
    }
  ],
  "PostToolUse": [
    {
      "matcher": "<도구명1>|<도구명2>",
      "hooks": [{ "type": "command", "command": "bash ~/.claude/hooks/<스크립트>.sh" }]
    }
  ]
}
```

## 현재 등록된 훅

### PreToolUse — `pre-commit-doc-check.sh`

- **matcher**: `Bash`
- **목적**: 소스 파일 커밋 시 도메인별 개발문서 (FE·BE) 존재 여부 확인
- **동작**:
  - FE 소스 → `<docs-dir>/<domain>/FE_*.md` 유무 검사
  - BE 소스 → `<docs-dir>/<domain>/BE_*.md` 유무 검사
  - 문서 없음 → `exit 2` (Claude에게 경고, 커밋 차단 안 함)
  - 문서 있지만 미스테이징 → 갱신 여부 경고
- **통과 조건**: git commit 아닌 명령 / `--amend` `--allow-empty` 옵션 / `docs:` 접두사 커밋

> 적용 소스 경로와 docs 경로는 프로젝트 레포의 `.claude/hooks-config.sh`로 설정한다 (팀 공유 가능).

### PostToolUse — `check-source-doc.sh`

- **matcher**: `Edit|Write`
- **목적**: 소스 파일 수정 직후 FE·BE 문서 작성·갱신 유도
- **동작**:
  - FE 소스 수정 → `<docs-dir>/<domain>/FE_*.md` 유무 확인
  - BE 소스 수정 → `<docs-dir>/<domain>/BE_*.md` 유무 확인
  - 문서 없음 → 예상 경로와 함께 작성 요청 출력
  - 문서 있음 → 신선도 날짜·이력 갱신 안내

> 적용 소스 경로 패턴은 프로젝트 레포의 `.claude/hooks-config.sh`로 설정한다 (팀 공유 가능).

## 훅 파일 위치

```
~/.claude/hooks/
├── pre-commit-doc-check.sh   ← PreToolUse:Bash — 커밋 전 문서 확인
└── check-source-doc.sh       ← PostToolUse:Edit|Write — 소스 수정 후 문서 유도
```

## exit 코드 규칙

| exit 코드 | 의미 |
|-----------|------|
| `0` | 정상 통과 |
| `2` | Claude에게 경고 메시지 전달 (non-blocking) |
| `1` | 오류 (blocking — 도구 실행 차단) |

## 새 프로젝트에 훅 적용 시

1. 훅이 없으면: `bash 하네스/install.sh --hooks` — 스크립트 복사·실행 권한·settings.json 자동 등록
2. `/project-init` 실행 → 소스 경로 자동 분석 → `.claude/hooks-config.sh` 자동 생성 (별도 수동 설정 불필요)
3. 수동 조정이 필요한 경우: `Claude Code/훅_스크립트_전문.md` 참조
